Live AI Brain Health

This is the primary live-health path in server mode. It uses the existing runtime artifacts already produced by the dashboard suite and the ops-status report, without adding any new control surface.
Use this section first when you want the current runtime view rather than historical or manually loaded demo artifacts.
  • Live snapshot reads `TemporaryQueue/ai_brain_metrics_live.json` written by the metrics watcher.
  • Ops status reads `TemporaryQueue/ops_status.json` written by `scripts/ops_status_report.py` and can refresh through `/api/ops_status`.
  • Analysis mode reuses the current live and near-live artifact files only: live health, activity scaling, hardware preflight, and the latest eval-backed scorecard.
  • Operator readiness separates global runtime state from current-record readiness evidence such as guard heartbeat freshness, signal source, process observation, and measurement/schema trace status.
  • Workflow pie payloads should be interpreted by source path and `generated_at`; the `TemporaryQueue` payload is the live attachment surface, while retained copies may differ.
  • Historical metrics, adversarial reports, and scorecards remain available below as secondary inspection surfaces.

              - Dashboard reads TemporaryQueue/activity_scaling_latest.json for latest+delta and can fall back to the history JSONL when the latest file has not been generated yet
